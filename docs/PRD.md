## What Changed

The older PRD and master-plan documents mixed:

- current local-first PWA reality
- future SaaS and extension vision
- execution planning

That made it too easy for future agents to overstate what already exists.

The new product package fixes that by separating:

- current reality
- target-state product strategy
- implementation-ready epic packets
- AI operating rules
- ClickUp and GitHub governance
