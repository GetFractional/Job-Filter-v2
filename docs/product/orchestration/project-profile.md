# Job Filter Project Profile

Last updated: 2026-03-12

## Product Thesis

Job Filter is a truthful, experiment-driven job-search workspace for serious operators. The wedge is fewer, better-fit roles, grounded positioning, clearer decision quality, and reusable job-search leverage without turning the product into an auto-apply machine.

## Foundation Series Spine

The Foundation Series is the design and messaging spine for Job Filter:

1. Market Intelligence
2. Brand Strategy
3. Shared Visual and Product System
4. Website and Public Funnel
5. Activation and Core App
6. Lifecycle Messaging and Email System

Use the governing packet and active chapter-owner packet before loading broader docs.

## Locked Program Decisions

- Entry model = `Free tier`
- Broader MVP app IA = design-locked now
- Auto-apply = future-state only

## Primary ICP

- experienced operator, builder, or functional leader
- multi-lane search
- high trust threshold
- willing to do focused setup work if it creates reusable leverage

## Source Ladder

1. verified ClickUp task packets and read-after-write receipts
2. current repo reality
3. governing packet and active packet
4. canonical docs under `docs/product/`
5. thin wrappers and legacy references

## Thread Model

- one permanent lead thread
- one active coding thread
- no build without a packet
- one writer per packet

## Delivery Rules

- governance first, implementation second
- do not let website work redefine activation or app structure
- do not let implementation packets reopen locked program decisions
- use local memory only as supporting recall, never as source of truth over packet or tracker state

## Monetization Frame

`Free tier` is the entry model. Higher-value paid framing should remain tied to leverage, strategy, and premium outputs, not to vague AI convenience claims.

## Non-Goals

- no current-state auto-apply promise
- no packetless implementation
- no multi-writer packet execution
- no governance drift between repo docs and ClickUp ownership
