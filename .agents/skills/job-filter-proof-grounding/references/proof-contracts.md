# Proof Contracts

## Purpose

The Proof Library is the source of truth for approved claims that can safely power downstream assets.

## Core Expectations

- statuses must be explicit
- unresolved proof must not silently auto-flow into assets
- lineage should remain traceable when proof is reused
- current-role handling must not create false date data
- extracted story fragments remain suggestions until approved
- `used`, `missing`, and `excluded` stay visible where rationale is shown

## Important Surfaces

- profile and guided story extraction
- import draft building
- proof save path
- settings review
- digital resume review/edit
- assets
- application tracking
- Why this draft / rationale views
- Q&A
- scoring
