# Copy System

## Core Jobs

Every screen should answer:

1. What is happening?
2. Why does it matter?
3. What happens next?
4. If trust is relevant here, what was used, missing, or excluded?

## Tone

- Warm
- Grounded
- Clear
- Premium without theatrics

## Prefer

- Target fewer roles that you can credibly win.
- Build one job profile. Use it to drive better-fit applications.
- Review what was used, missing, or excluded before you accept a draft.
- Build my job profile.
- See how it works.
- Continue setup.
- Save and continue.
- Use selected job.
- Accept draft.
- No resume? Start manually.
- Suggestions stay visible until you approve them.

## Avoid

- Build my profile on public surfaces unless the packet explicitly reopens it.
- Highlight
- Outcome when “Result” is clearer
- Rejected when “Excluded from use” is safer
- Parser, raw source, normalized source in the main path
- Source of truth and proof-grounded as hero or CTA language
- Dream job or hired faster as locked product copy without proof
- Generic “faster, smarter, better” claims without mechanism

## Belief Shift

- From: “This is setup work.”
- To: “This creates reusable leverage that helps me target better roles, shape stronger assets, and track my search with more control.”
